// Capabilities

What's on the Grid

⚡

Two-Player Local

Go head-to-head on the same keyboard. Player A uses WASD, Player B claims the arrow keys.

💥

Real-Time Collision

Wall hits, trail collisions, self-intersections, and simultaneous crashes — all detected at ~75 FPS.

🔄

Instant Replay

Hit ENTER or SPACE after any round ends and the arena resets. No restarts, no friction.

🎨

Visual HUD

Color-coded cycles, live heads-up display, splash screen intro, and a clean end-game results overlay.

🏗️

Clean OOP Architecture

Eight focused classes — each with a single responsibility. Built to extend: add AI, power-ups, or sound.

⚙️

Fully Configurable

Tune speed, arena size, trail thickness, collision radius, and frame delay in one central constants file.

// Rules of the Grid

01

Your cycle moves continuously — you can only steer, never stop.

02

180° U-turns are blocked. Plan your moves ahead.

03

Touch a wall, opponent trail, or your own trail and you lose.

04

If both players crash at the same time, it's a draw.

05

Last cycle surviving wins the round. Press ENTER to run it again.

// Local Setup

Run it Yourself

Step 01 — Clone the Repository

Download the project via Git or grab the ZIP from GitHub.

# Clone via HTTPS
git clone https://github.com/fardinfaruqi/TRON-Lightcycle-Game--Two-Player
cd "TRON-Lightcycle-Game--Two-Player"

Step 02 — Requirements

No external packages needed. The turtle module ships with Python 3.7+.

# Verify your Python version
python --version
# Should output Python 3.7 or higher

Step 03 — Launch

Run the main entry point from your terminal.

python Tron_Lightcycle.py
# Or: python3 Tron_Lightcycle.py

// Tuning

Configuration

All settings live in classes/constants.py — edit and replay immediately.

Constant Default Description
WIDTH, HEIGHT800 × 600Arena dimensions in pixels
STEP5Movement speed (pixels per frame). Raise for faster cycles.
FRAME_DELAY0.013Frame timing — lower values push toward higher FPS
TRAIL_THICKNESS3Width of the light trail in pixels
TRAIL_SAMPLE4Record a trail point every N frames (performance tuning)
COLLISION_RADIUS6Detection radius in pixels around each cycle
GRACE_FRAMES30Frames of immunity before self-collision is checked

// Code Structure

Architecture

Eight focused classes, each with a single clear responsibility. Extend any one of them without touching the others.

game.py

Game

Main orchestrator. Runs the game loop — init, update, collision check, render.

player.py

Player

Manages position, heading, color, trail recording, and direction queuing.

arena.py

Arena

Defines game boundaries, draws the border, validates positions within bounds.

collision_detector.py

CollisionDetector

Detects wall hits, opponent trail hits, and self-collisions with a grace period.

input_handler.py

InputHandler

Binds keyboard inputs to player actions. Keeps input logic separate from game logic.

ui_renderer.py

UIRenderer

Splash screen, HUD, end-game results. All text and graphics in one place.

game_state.py

GameState

Tracks game status (running / game_over / winner) and the frame counter.

constants.py

Constants

Central config for arena size, speed, collision radius, FPS and more.
